# 종합소득세 자동화 시스템 가이드 (2026 귀속)

---

## 1. 시스템 개요

```
┌─────────────────────────────────────────────────────────────────┐
│              종합소득세 신고 자동화 파이프라인                    │
│                                                                 │
│  [홈택스 자동수집]  →  [파싱/정리]  →  [작업준비]  →  [검증]   │
│                                                                 │
│  종소세안내문 PDF       parse_anneam     print_sheet.py         │
│  지급명세서 PDF    →    parse_jibup  →   jakupan_gen.py    →   교차검증│
│  간이용역소득 xlsx      parse_ganyi      summary_excel.py       │
│  전년도소득세 xls       parse_prev                              │
│  부가세신고내역 xlsx    parse_vat                               │
│                                                                 │
│  대상: 216명 / 기준연도: 2025 귀속 / 운영환경: Windows + NAS   │
└─────────────────────────────────────────────────────────────────┘
```

### 처리 흐름 요약

| 단계 | 스크립트 | 담당 | 출력 |
|------|---------|------|------|
| 1. 안내문 일괄 다운로드 | `종합소득세안내문조회.py` | 봇 | `종소세안내문_*.pdf` |
| 2. 지급명세서 일괄 다운로드 | `jipgum_batch.py` | 봇 | `지급명세서/*.pdf` |
| 3. 216명 요약 엑셀 | `summary_excel.py` | 자동 | `output/종소세요약.xlsx` |
| 4. A4 작업준비 시트 | `print_sheet.py` | 자동 | `작업준비_{이름}.xlsx` |
| 5. 작업판 자동입력 | `jakupan_gen.py` | 자동 | `작업판_{이름}.xlsx` |
| 6. 교차검증보고서 | `crosscheck_report.py` | 자동 | `검증보고서_{이름}.xlsx` |
| 7. 최종 세액 안내 | — | 세무사 | — |

## 2. 파일 구성 + 설치

### 주요 스크립트

```
F:\종소세2026\
├── 종합소득세안내문조회.py   # 홈택스 Playwright 자동화 (안내문+지급명세서+부가세)
├── jipgum_batch.py           # 지급명세서 일괄 PDF 다운로드
├── parse_to_xlsx.py          # 파싱 함수 모음 (parse_anneam, parse_prev, parse_vat)
├── summary_excel.py          # 216명 1행 요약 엑셀 생성
├── print_sheet.py            # A4 작업준비 시트 생성 (①②③④)
├── jakupan_gen.py            # 작업판 자동입력 + 작업준비시트 통합
├── crosscheck_sample.py      # 파싱 정확도 샘플 교차검증
├── config.py                 # 경로 설정 (SEOTAX_ENV: nas/local)
└── gsheet_writer.py          # 구글시트 연동
```

### 설치

```bash
# Python 패키지
pip install openpyxl xlrd pdfplumber gspread playwright

# Playwright 브라우저
playwright install chromium

# 환경변수 (NAS 연결 시)
set SEOTAX_ENV=nas
```

### 실행 방법

```bash
# 1명 작업준비 시트
python print_sheet.py 홍길동 800101

# 1명 작업판 통합 생성
python jakupan_gen.py 홍길동 800101

# 216명 전체
python jakupan_gen.py --all

# 파싱 샘플 교차검증
python crosscheck_sample.py
```

## 4. 파일 자동 인식 키워드

| 파일 종류 | 탐색 패턴 | 위치 |
|---------|---------|------|
| 종소세 안내문 PDF | `종소세안내문_*.pdf` | 고객 폴더 루트 |
| 전년도 소득세 | `전년도종소세신고내역.xls` | 고객 폴더 루트 |
| 부가세 신고내역 | `부가세신고내역_*.xlsx` | 고객 폴더 루트 |
| 지급명세서 PDF | `지급명세서/*.pdf` | 지급명세서 하위폴더 |
| 간이용역소득 | `간이용역소득/*.xlsx` | 간이용역소득 하위폴더 |
| 고객 폴더 | `{성명}_{생년월일6자리}` | `고객\` 하위 |

**간이용역소득 xlsx 컬럼 키워드** (행0 헤더 기준):

| 컬럼명 | 내용 |
|--------|------|
| `지급명세서 종류` | 간이지급명세서(거주자의 사업소득) 등 |
| `사업자(주민)등록번호` | 징수의무자 사업자번호 |
| `징수의무자` | 회사명 |
| `업종구분` | 자문/고문, 학원강사 등 |
| `총지급액` | 수입금액 |
| `소득세` | 원천징수 소득세 |
| `지방소득세` | 원천징수 지방소득세 |

**부가세 xlsx 키워드** (현재 홈택스 다운로드 파일):

| 키워드 | 탐지 방법 |
|--------|---------|
| `매출액` | 행[0] 포함 여부 → 매출 추출 |
| `매입액` | 행[0] 포함 여부 → 매입 추출 |
| `납부` / `환급` | 행[0] 포함 여부 → 납부세액 추출 |

> ⚠️ 현재 부가세 xlsx 파일이 헤더행(1행)만 저장되는 버그 있음. 재다운로드 필요.

## 5. 검증 항목 14개

| # | 구분 | 검증 내용 | 조건 (FAIL) |
|---|------|---------|-----------|
| 1 | 수입금액 일치 | 안내문 수입합계 = 지급명세서 합계 | 차이 > 1,000원 |
| 2 | 수입금액 일치 | 지급명세서 = 간이용역소득 합계 | 차이 > 1,000원 |
| 3 | 수입금액 추이 | 전년 대비 증감률 | ±50% 이상 |
| 4 | 소득금액 추이 | 3년간 소득금액 변동 | 급격한 역전 |
| 5 | 원천징수 확인 | 기납부세액 = 지급명세서 소득세 합계 | 차이 > 10,000원 |
| 6 | 경비율 적정성 | 신고 경비율 vs 업종 단순경비율 | 단순율 초과 의심 |
| 7 | 소득율 적정성 | 신고 소득율 vs 수입구간별 적정소득율 | GAP > 5,000만 |
| 8 | 기장의무 확인 | 안내문 기장의무 vs 신고 방법 일치 | 불일치 |
| 9 | 업종코드 확인 | 안내문 업종코드 vs 신고서 업종코드 | 불일치 |
| 10 | 사업자번호 확인 | 지급명세서 사업자번호 vs 간이용역 사업자번호 | 불일치 |
| 11 | 부가세 연동 | 부가세 매출 vs 종소세 수입금액 비율 | 비율 이상 |
| 12 | 전년도 세액 비교 | 전년 납부세액 vs 올해 납부세액 | ±200% 이상 |
| 13 | 중간예납 확인 | 안내문 중간예납 vs 기납부 반영 여부 | 누락 |
| 14 | 환급 여부 | 납부할총세액 음수 | 음수 → 환급 안내 |

## 6. 결과 해석 기준

| 판정 | 기준 | 조치 |
|------|------|------|
| ✅ PASS | 모든 검증 통과 | 세액 안내 진행 |
| ⚠️ WARN | 경미한 차이 (임계값의 50% 이내) | 세무사 확인 후 진행 |
| ❌ FAIL | 임계값 초과 or 필수항목 불일치 | 반드시 재검토 후 진행 |
| ➖ SKIP | 자료 없음 (부가세 등) | 수동 조회 후 진행 |

> FAIL 항목이 1개라도 있으면 고객 세액 안내 전 세무사 최종 확인 필수.

| 안내문 기장의무 | `parse_anneam()` | `parse_to_xlsx.py` L157 | `기장의무\s+(.+?)\s+추계시` |

| 지급명세서 PDF 파싱 | `parse_jibup_pdf()` | `print_sheet.py` | regex: `①사업자등록번호`, 데이터행 패턴 |

| 작업판 입력셀 | `fill_puri()` | `jakupan_gen.py` | `CLEAR_CELLS` 목록 + 셀주소 매핑 |

## 8. 알려진 제약사항

| 제약 | 내용 | 회피방법 |
|------|------|---------|
| 스캔 PDF | pdfplumber 텍스트 추출 불가 | 수동 입력 또는 OCR 도구 별도 사용 |
| 부가세 xlsx 버그 | 다운로드 시 헤더행(1행)만 저장됨 | 재다운로드 필요 (download_vat 함수 디버그) |
| 안내문 p.3 파싱 미완성 | 3년간 신고상황 자동 추출 아직 없음 | parse_anneam 확장 필요 |
| 큰 숫자 줄바꿈 | PDF에서 1억이 1천만으로 파싱될 수 있음 | `총계` 다음 행까지 합산 처리 (L167~174) |
| 복수 사업자 업종코드 | jakupan_gen 현재 첫 번째만 입력 | 다중 사업장 루프 처리 필요 |
| stdout 이중래핑 | parse_to_xlsx import 시 충돌 | `_saved_stdout` 패턴 반드시 유지 |
| 60초 타임아웃 지급명세서 | 행수 7~9개 고객 다운로드 실패 | 필요 시 수동 처리 (6명 확인됨) |
| 작업판 복식부기 | jakupan_gen 현재 간편장부 템플릿만 지원 | 빈양식_복식부기.xlsx 연동 필요 |
